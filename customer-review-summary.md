## Feedback received

- The customer asked how receipt data is currently collected into the database and noted that the receipt-scanning algorithm has not been implemented yet — flagged as a priority for an upcoming sprint.
- The customer emphasized that before generating recipes, the app should first collect user parameters (goal: lose weight / maintain weight / gain weight, plus weight and height) so that recipe and meal suggestions can be personalized.
- The customer noted that the UX/UI design looks close to done, but considered it less critical at this stage compared to the backend/AI functionality.
- The customer pointed out that only 5 API endpoints have been implemented so far and considered this insufficient. They asked the team to extend the API so it can support the AI/recipe-generation features across all planned platforms (website, Telegram bot, mobile app).
- The customer asked to review the UX/UI prototype separately and will provide feedback via private message or the team's group chat.
