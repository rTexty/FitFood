# Definition of Done (DoD) - FitFood

An item is considered "Done" and ready for inclusion in a release when it meets the following criteria:

## 1. Code Quality & Standards
* Code compiles and runs without critical errors in the local environment.
* The branch structure aligns with the repository governance policies.
* No temporary or untracked environment data (e.g., local `.venv`, local databases) is committed.

## 2. Testing & Verification
* Core edge-cases for the implemented features are verified.
* Database models match the migration or schema baseline.

## 3. Documentation & Process
* The corresponding GitHub Issue/User Story is updated with story points.
* Any changes to the core application paths are updated in the requirements index if needed.
