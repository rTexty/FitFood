# FitFood — User Stories

This document lists user stories for the FitFood MVP, prioritized using the MoSCoW method. Each story follows the format: *As a [type of user], I want to [perform an action], so that [I achieve a goal/value]*.

---

## Must Have (MVP v1)

**US-01**
As a user, I want to add products to my kitchen inventory, so that I can keep track of what I have.

**US-02**
As a user, I want to scan a receipt QR code, so that products can be added automatically to my inventory.

**US-03**
As a user, I want to see product expiration dates, so that I can use food before it expires.

**US-04**
As a user, I want to receive notifications about products that are close to expiring, so that I can reduce food waste.

**US-05**
As a user, I want to set a fitness goal (weight loss, maintenance, or weight gain), so that the app can create a suitable nutrition plan.

**US-06**
As a developer, I want a public API for the application, so that the system can be integrated with a website, Telegram bot, or mobile app.

---

## Should Have

**US-07**
As a user, I want the app to recommend recipes based on the ingredients available in my kitchen, so that I can use my existing products efficiently.

**US-08**
As a user, I want a weekly meal plan generated by AI, so that I can follow my nutrition goals easily.

**US-09**
As a user, I want to view nutritional information for recommended recipes, so that I can make informed food choices.

---

## Could Have

**US-10**
As a user, I want to upload a photo of my meal, so that AI can estimate calories automatically.

**US-11**
As a user, I want to track how much of each product I have left (by weight or count), so that I know when to buy more.

---

## MVP v1 Scope

Based on the MoSCoW prioritization, the MVP v1 will include user stories **US-01 through US-06**. These cover the two core problems identified by the customer: product/expiration tracking and recipe matching, plus the foundational API required for cross-platform access.
