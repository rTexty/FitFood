# LLM Usage Report — Week 4

## Tools used

- **Claude (Anthropic)** — used by Nurmuhammet (QA/Reporter)

## How AI tools were used

### Documentation and reporting
Claude was used to assist with the following documentation tasks during Week 4:

- **Customer review transcript** (`reports/week4/customer-review-transcript.md`): The raw meeting transcript was cleaned for readability using Claude — filler words, false starts, and redundant repetitions were removed without changing the meaning. Timestamps were formatted and speaker roles were substituted for names.

- **Customer review summary** (`reports/week4/customer-review-summary.md`): Claude helped structure the customer feedback into a clear summary document, including the list of requested changes, action points table, and approval status.

- **Sprint retrospective** (`reports/week4/retrospective.md`): Claude assisted in drafting the retrospective based on the team's Sprint experience, covering what went well, what did not go well, and concrete action points.

- **Sprint reflection** (`reports/week4/reflection.md`): Claude assisted in drafting the reflection document, including learning points, validated assumptions, friction and gaps, and planned responses for the next Sprint.

## Original effort verification

All core content in the above documents is based on real team activity, the actual customer review session, and genuine Sprint outcomes. Claude was used as a writing and structuring assistant — all decisions, judgments, and content were reviewed and approved by the team member responsible for the artifact. No filler or generic AI-generated content was submitted without review.

## AI tools in product implementation

AI tools including LLM assistants were also used to support coding, database work, and product implementation during Week 4. Given the scale of the codebase, AI-assisted code generation was used alongside manual development. All generated code was reviewed, tested, and integrated by the respective team members.
